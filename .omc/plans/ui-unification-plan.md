# UI 통일화 작업 계획

## 개요

**목표**: 트레이너/회원 앱 전체에서 일관된 디자인 시스템 적용
**범위**: 색상, 폰트, 간격, 컴포넌트 스타일 통일 (기능별 레이아웃은 유지)
**기준 화면**: 홈 화면 (현재 가장 잘 정돈된 디자인)
**도구**: Stitch MCP (이미 설정됨)

---

## 현재 디자인 시스템 분석

### 이미 잘 구축된 부분
| 항목 | 위치 | 상태 |
|------|------|------|
| 디자인 토큰 | `lib/core/theme/app_tokens.dart` | spacing, radius, shadow, animation 정의됨 |
| 색상 테마 | `lib/core/theme/app_theme.dart` | Flex Color Scheme + Pretendard 폰트 |
| 공통 컴포넌트 | `lib/presentation/widgets/common/` | AppButton, AppCard, AppTextField |
| 애니메이션 | `lib/presentation/widgets/animated/` | 일관된 진입 애니메이션 |
| 스켈레톤 로더 | `lib/presentation/widgets/skeleton/` | Shimmer 기반 로딩 |

### 통일이 필요한 부분
1. **일부 화면에서 직접 스타일 하드코딩** - 공통 컴포넌트 미사용
2. **카드 스타일 불일치** - 일부 화면에서 다른 border, shadow 사용
3. **간격 불일치** - 토큰 대신 임의의 숫자 사용
4. **애니메이션 불일치** - 일부 화면에서 애니메이션 없음
5. **PAL 로고 pill** - 제거 예정

## 작업 원칙

1. **기존 기능 유지**: UI만 수정, 로직 변경 없음
2. **점진적 적용**: 한 화면씩 수정 후 테스트
3. **토큰 사용 강제**: 매직 넘버 금지
4. **다크모드 필수**: 모든 변경에 다크모드 대응 포함
5. **최소 변경**: 필요한 부분만 수정, 과도한 리팩토링 지양

---

## Stitch MCP 활용 계획

1. **디자인 토큰 문서화**: 현재 토큰을 Stitch에 등록
2. **컴포넌트 갤러리**: 공통 위젯 사용 예시 문서화
3. **화면별 스펙**: 각 화면의 사용 컴포넌트 명시
4. **변경 추적**: 통일화 전/후 비교 이미지 저장
