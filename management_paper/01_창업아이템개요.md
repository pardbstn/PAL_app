# 1. 창업아이템 개요(Business Overview)

---

## 1-1. 아이템 개요

### 아이템명

**PAL (팔)** - Progress, Analyze, Level-up

### 핵심 가치 (One-liner)

> "PT 데이터가 쌓일수록 똑똑해지는 AI. 트레이너는 PT만, 나머지는 PAL이."

### 서비스 정의

PAL은 **PT 과정에서 자연스럽게 축적되는 데이터**를 기반으로 **AI가 예측하고 인사이트를 도출**하는 **트레이너 전용 회원관리 플랫폼**입니다.

| 구분 | 내용 |
|------|------|
| **서비스명** | PAL (팔) - Progress, Analyze, Level-up |
| **슬로건** | "기록하고, 분석하고, 성장하다." |
| **핵심 타겟** | PT 트레이너 (B2B2C) |
| **플랫폼** | 모바일 앱 (Android/iOS) + 웹 대시보드 |
| **비즈니스 모델** | Freemium SaaS (월 0원~3만원) |

---

### 왜 GPT가 아닌 PAL인가?

#### ChatGPT로는 불가능한 것

"GPT한테 물어보면 되지 않나요?"라는 질문에 대한 답변입니다.

| 구분 | ChatGPT 직접 사용 | PAL |
|------|-------------------|-----|
| **데이터 축적** | 없음 (매번 새로 입력) | 자동 축적 (수개월~수년) |
| **개인화 수준** | 단발성 질문 기반 | 장기 데이터 기반 깊은 개인화 |
| **예측 가능성** | 불가능 | 체중 예측, 이탈 예측, 성과 예측 |
| **관리 효율성** | 회원 20명 데이터 매번 입력? | 자동 수집, 원클릭 분석 |
| **인사이트 품질** | 일반적 조언 | 실제 데이터 기반 구체적 인사이트 |

**핵심 차이점:**

```
┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│  ChatGPT: "다이어트하려면 유산소 운동을 하세요"                      │
│                    ↓ 일반적 조언                                    │
│                                                                     │
│  PAL:     "김민수 회원님, 지난 8주간 데이터 분석 결과               │
│           하체 운동 시 체지방 감량 속도가 1.5배 빠릅니다.           │
│           현재 속도 유지 시 4주 후 목표 체중 75kg 도달 예상.        │
│           단, 최근 2주간 단백질 섭취가 25% 부족하니                 │
│           식단 조절이 필요합니다."                                  │
│                    ↓ 실제 데이터 기반 개인화 인사이트               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**트레이너가 GPT를 직접 사용한다면:**
- 회원 20명의 수개월치 운동/체중/식단 데이터를 매번 복사-붙여넣기?
- 각 회원별 이탈 위험도를 어떻게 계산?
- 체중 예측은 어떻게?

**PAL은 이 모든 것을 자동으로 처리합니다.**

### 개발 동기 및 배경

#### 창업자의 3년간 PT 회원 경험

창업자는 3년간 PT를 받으며 트레이너들의 업무 현실을 직접 목격했습니다.

> "제 트레이너님은 항상 수업 시작 전에 노트북을 펴고 뭔가를 정리하셨어요.
> 물어보니 회원 데이터를 엑셀에 입력하는 거라고 하더군요.
> 수업 중에도 종종 '다른 회원분이 어떻게 하셨더라...' 하며 기억에 의존하셨고요.
> 이 시간에 저한테 운동 하나라도 더 알려주시면 좋겠다고 생각했어요."

#### PT 트레이너의 현실

- 국내 PT 트레이너 약 **2.5만명** (문화체육관광부, 2019)
- 연간 PT 이용자 약 **80~100만명** (KB금융지주경영연구소, 2020)
- 회원 1명당 커리큘럼 작성에 **30분~1시간** 소요
- 주당 **7~20시간**을 행정업무에 투입
- 대부분 **엑셀, 노션, 종이**로 수동 처리

#### 트레이너가 겪는 5가지 핵심 문제

| 문제 | 현황 | 결과 |
|------|------|------|
| **커리큘럼 작성** | 수동 작성 30분~1시간/회원 | 트레이너 번아웃 |
| **데이터 관리** | 엑셀/노션에 수기 입력 | 기록만 하고 활용 못함 |
| **이탈 감지** | 파악 방법 없음 | 회원 이탈 → 수입 감소 |
| **회원 동기부여** | 변화 시각화 자료 없음 | PT 조기 종료 |
| **계약서 관리** | 종이 계약서 보관 | 분실/법적 분쟁 위험 |

#### 시장의 공백

기존 PT 관리 앱들은 **'기록'만 가능**합니다.

| 기존 솔루션 | PAL |
|-------------|-----|
| 데이터를 **입력**만 함 | 데이터를 **축적**하고 **분석**함 |
| 그래프는 **수동** 제작 | 그래프 **자동** 생성 |
| 예측 **불가능** | AI가 체중/이탈/성과 **예측** |
| 인사이트 **없음** | 축적 데이터 기반 **인사이트 자동 생성** |

**PAL은 이 공백을 "데이터 축적 + AI 분석"으로 채웁니다.**

## 1-2. 창업아이템 차별성

### 핵심 경쟁력: 데이터 축적 기반 AI

#### PAL의 3가지 핵심 경쟁력

| 경쟁력 | 설명 | 효과 |
|--------|------|------|
| **1. 자동 데이터 축적** | PT 과정에서 자연스럽게 데이터 수집 | 사용자 입력 부담 최소화 |
| **2. 시간 기반 가치 증가** | 데이터가 쌓일수록 예측 정확도 향상 | 전환 비용 증가, Lock-in 효과 |
| **3. 개인화 인사이트** | 축적 데이터 기반 구체적 인사이트 | GPT 대비 압도적 개인화 수준 |

---

### 자연스럽게 쌓이는 데이터 (사용자 입력 부담 최소화)

#### 자동 수집 데이터 (입력 불필요)

| 데이터 | 수집 방식 | 활용 |
|--------|----------|------|
| **출석률** | PT 일정 완료/노쇼/취소 자동 기록 | 이탈 예측, 습관 분석 |
| **운동 볼륨** | 커리큘럼 완료 시 자동 집계 (세트×무게×횟수) | 성과 추적, 정체기 감지 |
| **PT 진행률** | 잔여 횟수 자동 계산 | 재등록 시점 예측 |
| **수업 패턴** | 요일별/시간대별 선호도 자동 분석 | 최적 스케줄 추천 |
| **메시지 응답률** | 채팅 데이터 기반 소통 빈도 | 이탈 징후 감지 |
| **커리큘럼 난이도** | AI 생성 커리큘럼 강도 추이 | 성장 속도 분석 |

#### 반자동 수집 데이터 (최소 입력)

| 데이터 | 입력 방식 | 소요 시간 |
|--------|----------|----------|
| **체중** | 숫자만 입력 | 1초 |
| **식단** | 사진 1장 촬영 → AI 자동 분석 | 3초 |
| **인바디** | 수치 입력 또는 사진 OCR | 30초 |

**핵심: 트레이너와 회원이 "기록한다"는 느낌 없이 데이터가 축적됩니다.**

### 축적 데이터 기반 AI 인사이트

#### 트레이너용 인사이트 (7가지)

| # | 인사이트 | 활용 데이터 | 예시 | 가치 |
|---|----------|-------------|------|------|
| 1 | **이탈 위험 예측** | 출석률, 메시지 응답률, 체중 정체 | "김민수 회원 이탈 위험도 78% - 2주간 출석률 40%↓" | 선제 대응 → 회원 유지 |
| 2 | **재등록 가능성** | 목표 달성률, 만족도 지표 | "이수진 회원 재등록 가능성 85% - 목표 92% 달성" | 영업 우선순위 설정 |
| 3 | **최적 커리큘럼** | 운동별 성과, 선호도 | "박지영 회원은 하체 운동 시 감량 속도 1.5배" | 성과 극대화 |
| 4 | **정체기 감지** | 체중/체성분 추이 | "최민호 회원 4주째 정체 - 강도 20% 증가 필요" | 조기 대응 |
| 5 | **성과 랭킹** | 전체 회원 데이터 | "이번 달 감량 TOP3: 1위 박지영(-2.3kg)" | 동기부여 자료 |
| 6 | **노쇼 패턴** | 요일/시간대별 노쇼율 | "월요일 오전 노쇼율 35% - 일요일 리마인더 권장" | 시간 효율화 |
| 7 | **최적 수업 시간** | 시간대별 운동 성과 | "김민수 회원은 저녁 7시 수업 시 볼륨 20%↑" | 스케줄 최적화 |

#### 회원용 인사이트 (6가지)

| # | 인사이트 | 활용 데이터 | 예시 | 가치 |
|---|----------|-------------|------|------|
| 1 | **체중 예측** | 체중 변화 추이 | "4주 후 목표 체중 75kg 도달 예상" | 목표 시각화 |
| 2 | **운동 성과** | 1RM 변화, 볼륨 증가 | "스쿼트 4주간 15kg↑! 다음 달 100kg 도전 가능" | 성취감 |
| 3 | **영양 밸런스** | 식단 기록 누적 | "단백질 25% 부족 - 닭가슴살 100g 추가 권장" | 행동 가이드 |
| 4 | **체성분 변화** | 인바디 기록 | "3개월: 체지방 -4.1kg, 골격근 +1.2kg" | 장기 성과 시각화 |
| 5 | **출석 습관** | 출석 패턴 | "출석률 85%로 상위 20%입니다" | 경쟁심 자극 |
| 6 | **컨디션 패턴** | 운동 성과 변화 | "월요일 운동 성과가 가장 좋음" | 자기 이해 |

### 기술적 차별성

#### AI 기능 4가지

| 기능 | 기술 | 입력 | 출력 | 성능 목표 |
|------|------|------|------|----------|
| **AI 커리큘럼** | GPT-4o | 회원 목표/경력 | N회차 운동 계획 | 30초 이내 생성 |
| **AI 체중 예측** | Prophet | 4주+ 체중 데이터 | N주 후 예상 체중 | 오차 ±2kg |
| **AI 인사이트** | GPT-4o + 패턴 분석 | 축적 데이터 | 개인화 인사이트 | 주 1회 자동 생성 |
| **식단 Vision** | GPT-4o Vision | 음식 사진 | 칼로리/영양소 | 인식률 80% |

#### 피드백 루프 (시간 기반 개선)

```
데이터 축적 → AI 예측 → 실제 결과 비교 → 오차 학습 → 예측 정확도 향상
     ↑                                                              ↓
     └──────────────────── 반복 ────────────────────────────────────┘
```

**예시:** 체중 예측 정확도
- 4주 데이터: 오차 ±3kg
- 12주 데이터: 오차 ±2kg
- 24주 데이터: 오차 ±1kg

### B2B2C 모델: 회원 유입 구조

#### 트레이너를 통한 회원 자연 유입

PAL은 **B2B2C 모델**을 채택합니다. 트레이너(B2B)가 PAL을 사용하면, 회원(C)은 **자연스럽게** 앱을 설치하게 됩니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    PAL B2B2C 유입 모델                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   트레이너 가입       회원 초대          회원 자연 유입              │
│   (PAL 사용 시작)  →  (앱 설치 링크)  →  (앱 설치 필수)             │
│                                                                     │
│   ┌─────────┐       ┌─────────┐       ┌─────────┐                 │
│   │트레이너1│ ────→ │ 회원 10명│       │         │                 │
│   └─────────┘       └─────────┘       │  CAC    │                 │
│   ┌─────────┐       ┌─────────┐       │  = 0원  │                 │
│   │트레이너2│ ────→ │ 회원 15명│       │         │                 │
│   └─────────┘       └─────────┘       └─────────┘                 │
│                                                                     │
│   트레이너 200명 × 회원 평균 10명 = 회원 2,000명 (마케팅비 0원)     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

#### 회원이 앱을 설치해야 하는 5가지 이유

| # | 설치 동기 | 상세 설명 | 대안 부재 |
|---|----------|----------|----------|
| 1 | **커리큘럼 확인** | 트레이너가 PAL에서 작성한 커리큘럼은 회원 앱에서만 확인 가능 | 카톡/문자로 따로 전송 안함 |
| 2 | **PT 예약/변경** | 수업 일정 예약, 변경, 취소가 앱에서만 가능 | 전화/문자 불가 |
| 3 | **전자 서명** | PT 계약서, 수업 완료 확인 서명이 앱에서만 가능 | 센터 정책상 필수 |
| 4 | **트레이너 소통** | 식단 피드백, 운동 질문 등 앱 내 DM으로 처리 | 다른 채널 사용 안함 |
| 5 | **변화 기록 확인** | 체중/체성분 변화 그래프, 운동 기록 열람 | 앱에서만 시각화 제공 |

**핵심: 트레이너가 PAL을 사용하면, 회원은 "선택"이 아닌 "필수"로 앱을 설치합니다.**

> **고객획득비용(CAC) = 0원**
> - 일반 B2C 앱: 설치당 2,000~5,000원
> - PAL: 트레이너 1명 획득 → 회원 10~15명 자동 유입

## 1-3. 사업화 방향

### 목표 시장

#### 시장 규모 (TAM-SAM-SOM)

| 구분 | 규모 | 산출 근거 |
|------|------|----------|
| **TAM** | **1.5조원** | 국내 PT 시장 (KB금융지주경영연구소, 2020) |
| **SAM** | **300억원** | PT 트레이너 SW 시장 (2.5만명 × 연 12만원) |
| **SOM** | **18억원** (3년차) | 유료 5,000명 × 월 3만원 × 12개월 |

#### 타겟 고객

| 단계 | 타겟 | 특징 | 목표 |
|------|------|------|------|
| **1단계** | 개인 트레이너 (프리랜서) | IT 친화적, 효율화 니즈 | 유료 500명 (1년차) |
| **2단계** | 소형 PT샵 (1~3인) | 비용 민감, 확장 니즈 | 유료 2,000명 (2년차) |
| **3단계** | 피트니스 센터 (B2B) | 조직 도입, 대량 계약 | 유료 5,000명 (3년차) |

| 티어 | 월 가격 | 회원 수 | 핵심 기능 |

| **Free** | **0원** | 5명 | 기본 기록, 그래프, DM |

### 사업화 로드맵

#### Phase 1: MVP 완성 + 검증 (1~3개월)

| 항목 | 내용 |
|------|------|
| **목표** | 핵심 기능 검증, 데이터 축적 시작 |

| **마케팅** | 트레이너 커뮤니티 바이럴 |
| **지표** | 트레이너 100명, 예측 정확도 80% |

#### Phase 3: 정식 출시 + 수익화 (7~10개월)

| 항목 | 내용 |
|------|------|
| **목표** | 정식 서비스 런칭, 매출 창출 |
| **개발** | 결제 시스템, 웹 대시보드 |
| **수익화** | Free/Basic/Pro 3티어 구독 |
| **지표** | 유료 전환율 10%, 월 매출 500만원 |
